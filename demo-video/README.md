# HomeCanvas demo video

A storyboard-driven product demo (under 60s, 1080p30) built with [Remotion](https://remotion.dev). Every visual is React/CSS/SVG motion graphics in the app's **Sarvam-inspired light theme** — cool light chrome, white cards, near-black navy ink, indigo accent, soft warm gradient washes, and custom thin-line iconography (`src/components/Icons.tsx`). Includes a real **CSS-3D dollhouse** (no WebGL) that the camera orbits, walks into, and "path-traces" tile by tile. No screenshots, no private data.

The rendered video lives at [`../docs/homecanvas-demo.mp4`](../docs/homecanvas-demo.mp4).

## Storyboard (8 scenes, <60s)

| # | Scene | Visual |
|---|---|---|
| 1 | `title` | Wordmark + tagline |
| 2 | `trace` | Walls tracing over the plan + live isometric 3D rising |
| 3 | `canvas3d` | CSS-3D dollhouse orbit → top → walk → tour (or your real walkthrough clip) |
| 4 | `edit` | Materials, style packs, stair rotate, pillar-delete warning |
| 5 | `ai` | Assistant chat: type → proposal → Apply |
| 6 | `photo` | Photo Mode tile-by-tile convergence (over your real render, if captured) |
| 7 | `audience` | Homeowners & buyers · Architects · Interior designers · Developers |
| 8 | `outro` | Local-first line + email CTA |

Scene lengths derive from the narration audio (`src/durations.json`), so re-recording narration re-times the video automatically.

## Narration

Generated by `scripts/generate-narration.mjs`:

- **Preferred — ElevenLabs** (expressive male Indian-English voice "Aashish", `eleven_multilingual_v2`): set `ELEVENLABS_API_KEY` in the environment. The key is **never stored in this repo**. Override the voice with `DEMO_VOICE_ID`.
- **Fallback — macOS `say`** (no key needed; robotic but free).

Either way the output is `public/narration/<id>.wav` + `src/durations.json`. Caption text lives in `src/narration.json`; the script has a pronunciation-override map (e.g. the email address is spoken, not spelled).

## Use your REAL app footage (penthouse cut — local only)

The committed video uses synthetic CSS visuals so it stays anonymized. To cut a
version with your **actual app UI** (3D walkthrough + a real Photo Mode render),
capture on a machine with WebGL2 (this is the one part that can't be generated
headlessly):

```
# in the app:  npm run dev  →  http://localhost:5173  →  "My Home"
# Cmd+Shift+5, record a few seconds of orbit / walk / tour
# click "Photoreal", let samples climb, then "Save photo" (PNG)
```

Drop the files into `public/captures/`, named by slot:

| File | Used by |
|---|---|
| `walkthrough.mp4` (or `walk.mp4` / `orbit.mp4` / `tour.mp4`) | the 3D-canvas scene |
| `photoreal.png` (or `photoreal.mp4`) | the Photo Mode scene |

Then:

```bash
npm run render:penthouse   # → out/homecanvas-penthouse.mp4
```

`public/captures/` and the penthouse render are **git-ignored** — your home's
footage never enters git. `scan-captures` also marks `src/captures.json`
skip-worktree so a populated map can't be committed. Any slot you don't capture
falls back to the synthetic scene, so partial captures still render.

## Rebuild it

```bash
cd demo-video
npm install
ELEVENLABS_API_KEY=... node scripts/generate-narration.mjs   # or omit the key for macOS say
npx remotion studio src/index.ts                             # live preview
npm run render                                               # out/homecanvas-demo.mp4
```
